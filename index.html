Lesson 3: OS Command Injection (CWE-78)

Understanding and preventing command injection attacks

What is Command Injection?

An attack where malicious commands are executed on the host operating system through a vulnerable application.

Attack Example:
Intended: LIST /home/[userName]
Attacker input: ;DELETE ALL FILES
Result: LIST /home/; DELETE ALL FILES

Why It's Dangerous

  • Full control of victim's machine
  • Privilege escalation if app runs as root
  • Can bypass authentication
Mitigation Strategies:
  • Use parameterized APIs (avoid shell execution)
  • Implement strict input validation (allowlist)
  • Escape special characters: ; & && | ||
  • Apply principle of least privilege

Lesson 4: SQL Injection (CWE-89)

Database attacks through improper input handling

What is SQL Injection?

A vulnerability where attackers inject malicious SQL code into queries, bypassing authentication and accessing/modifying data.

Classic Authentication Bypass:
Normal query: SELECT * FROM users WHERE username='bob' AND password='pass123'
Attacker input: ' OR 1=1 --
Result: SELECT * FROM users WHERE username='' OR 1=1 --' AND password=''
The attacker is now logged in as the first user (often admin)!

Consequences

  • Unauthorized data access (read sensitive records)
  • Data modification or deletion
  • Authentication bypass
  • Potential system compromise
Best Defense - Prepared Statements:

Send SQL structure first with placeholders, then data separately. Database treats input as data only, not executable code.


Additional Defenses:
  • Input validation with allowlists
  • Principle of least privilege for DB accounts
  • Avoid dynamic SQL in stored procedures

Lesson 5: Cross-Site Scripting (XSS - CWE-79)

Injecting malicious scripts into trusted websites

What is XSS?

An attack that injects malicious JavaScript into a trusted website, bypassing the Same-Origin Policy and executing in victims' browsers.

Three Types of XSS

  • Reflected XSS: Payload in URL, not stored on server (one-time attack via malicious link)
  • Stored XSS: Payload saved in database (affects all users viewing the page)
  • DOM-Based XSS: Client-side vulnerability in JavaScript DOM manipulation
Consequences:
  • Session/cookie theft (session hijacking)
  • Credential theft via keylogging
  • Website defacement
  • Malware distribution
  • Unauthorized actions on behalf of user
Prevention Methods:
  • Output encoding: Convert special chars (<, >, &) to entities (<, >, &)
  • Input validation: Enforce strict rules for user input
  • Sanitization: Use trusted libraries (OWASP Java HTML Sanitizer)
  • HTTPOnly cookies: Prevent JavaScript from reading cookies
  • Content Security Policy (CSP): Control which resources can load

Lesson 6: Path Traversal (CWE-35)

Escaping restricted directories to access sensitive files

What is Path Traversal?

An attack that manipulates file paths to access files outside the intended restricted directory.

The "../" Attack:
Intended path: /web/images/picture.jpg
Attacker input: ../../../../etc/passwd
Result: /web/images/../../../../etc/passwd → /etc/passwd
Attacker now accesses system password file!

Evasion Techniques

  • Percent encoding: %2e%2e%2f
  • Double encoding: %252e%252e%252f
  • Unicode: %u002e%u002e%u002f
  • NULL bytes: %00

Consequences

  • Read sensitive config files and credentials
  • Access application source code
  • Modify or delete critical files
  • Information gathering for further attacks
Defense Strategies:
  • Strict allowlists: Only accept pre-approved file names/paths
  • Path normalization: Remove ../ sequences before processing
  • Least privilege: Run app with minimal file access permissions
  • Execution isolation: Use chroot jails or containers

Lesson 7: Burp Suite

Industry-standard web security testing platform

What is Burp Suite?

An integrated platform for performing security testing of web applications. The Swiss Army knife of web security.

Core Components

  • Proxy: Intercepts HTTP(S) traffic between browser and server (most crucial component)
  • Repeater: Manually modify and resend individual requests for testing
  • Intruder: Automated attack tool for SQLi, XSS testing with payloads
  • Scanner: Automated vulnerability scanner (Professional only)
  • Sequencer: Analyzes session token randomness quality
Editions:
• Community (Free): Essential manual testing toolkit
• Professional (Paid): Advanced features + automated Scanner

Lesson 8: Network Traffic Analysis

Capturing, inspecting, and analyzing network data flow

What is Network Traffic Analysis?

The process of capturing, inspecting, and analyzing network data to understand ports, protocols, patterns, and identify suspicious activity.

OSI Model (7 Layers)

  • Layer 7 - Application: User-level protocols (HTTP, FTP)
  • Layer 6 - Presentation: Data formatting, encryption (SSL, TLS)
  • Layer 5 - Session: Connection management
  • Layer 4 - Transport: TCP/UDP communication
  • Layer 3 - Network: Routing & addressing (IP)
  • Layer 2 - Data Link: Frames between devices (Ethernet)
  • Layer 1 - Physical: Hardware-level signals

Common Analysis Tools

Command-Line: tcpdump, TShark, NGrep, tcpick
Graphical: Wireshark (most widely used)
Hardware: Network Taps, SPAN Ports

Analysis Process

  • 1. Ingest: Capture packets using tools
  • 2. Filter: Remove irrelevant traffic noise
  • 3. Analyze: Identify patterns and extract data
  • 4. Detect: Use IDS/IPS to alert on threats

Wireshark Essentials

Display Filters:
  • http.request.method == GET
  • ip.addr == x.x.x.x
  • tcp.port == 80
  • dns
Capture Filters:
  • host 192.168.1.10
  • port 80
  • not port 22

Lesson 9: Email Security Fundamentals

Email encryption, authentication protocols, and threat analysis

Email Security Principles

  • Confidentiality: Only authorized users can access email
  • Integrity: Email content is not altered
  • Availability: Email systems accessible when needed
  • Authentication: Verifies sender is legitimate
  • Non-repudiation: Sender cannot deny sending message

Email Encryption Standards

S/MIME: Uses digital certificates from CAs, built into enterprise email clients
OpenPGP: Uses public-key cryptography with Web of Trust model, decentralized

Common Email Threats

  • Phishing: Tricks users into giving information or installing malware
  • Spear Phishing: Targeted, personalized attacks
  • Email Spoofing: Forged sender addresses
  • BEC: Business Email Compromise for fraud
  • Malware: Delivered through attachments or links

Email Authentication Protocols

SPF (Sender Policy Framework): Checks if sending IP is authorized
DKIM (DomainKeys Identified Mail): Cryptographically signs email to verify integrity
DMARC: Policy telling email servers how to handle SPF/DKIM failures

Analysis Tools

  • Message Header Analyzers (visualize email path and authentication)
  • REMnux Tools: emldump, msg-extractor, msgconvert, mail-parser

Lesson 10: Introduction to Event Logs

Understanding system logs for security monitoring and incident response

What Are Event Logs?

Records of significant activities or occurrences recognized by a software system. They act as a digital footprint and critical audit trail for troubleshooting and security.

Why Logs Matter

  • Identify malicious activity: How intruders entered, what they did, evidence left behind
  • Digital footprint: Track logins, file access, network connections, errors
  • Essential information: Timestamp, source, event type, additional context

Common Log Types

Application Logs: App-specific errors, warnings
Audit Logs: User actions tied to compliance
Security Logs: Login/logout, access control changes
System Logs: Boot, shutdown, kernel messages
Network Logs: Traffic flow, connection attempts
Web Server Logs: URL requests, HTTP codes, user-agents

Log Correlation Power

Combining multiple logs reveals: What happened? When? Where? Who? Was it successful? What was the result?

Windows Event Logs

  • Format: Binary (.evt/.evtx), can be converted to XML
  • Location: C:\Windows\System32\winevt\Logs
  • Categories: System, Security, Application
  • Key Info: Level (severity), timestamp, source, Event ID, task category

Log Analysis Tools

Windows:
  • Event Viewer: GUI-based viewing and filtering
  • Wevtutil.exe: Command-line tool for automation
  • Get-WinEvent: PowerShell cmdlet with advanced filtering
SIEM Platforms: Azure Sentinel, Splunk, IBM QRadar (centralize, correlate, alert)

Important Security Event

Event ID 7045: New service installed (often indicates malware or persistence mechanism)
